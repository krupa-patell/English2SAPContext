The client begins by generating a random private key (sKc) which is a random integer between 0 and 2^256 -1. The client public key (pKc) is created from the private key using the X25519 Key Exchange method. The client initial key is also generated. 8 bytes of random data is generated called the DCID. The items produced from this event are the client_initial_key, server_initial_key. The first UDP Datagram is sent through the VPN channel in the form of an initial clientHello packet. This is encrypted with the client_initial_key. The server now generates a random private key (sKs). The server public key (pKs) is created from the private key using the X25519 Key Exchange method. The server initial key is generated. It receives 8 bytes of Destination Connected ID from the ClientHello packet, and HKDF method. The items produced from this are the client_initial_key, server_initial_key. The server now sends its initial packet, as well as the handshake packet, composed in one UDP Datagram through the VPN channel. This includes the serverHello. It also has the serverHandshake, which includes TLS certificate. This is encrypted with the server_initial_key. The server now calculates the server handshake keys. This uses the pKc and sKs to calculate the TLS_shared_secret. The equation is TLS_shared_secret = pKs^sKc = pKc^sKs = 'g'^(sKc*sKs). The TLS_shared_secret and SHA256 Hash(clientHello + serverHello) are put into the HKDF to produce client_handshake_key and server_handshake_key. The server now sends its server handshake finished packet in another UDP Datagram through the VPN channel. This includes the certificate_verify and serverHandshake_finished messages. This is encrypted with the server_handshake_key. The client now calculates its handshake keys. This uses the pKs, sKc to calculate the shared_secret. The shared_secret and SHA256 Hash(clientHello + serverHello) are put into key derivation operations to produce client_handshake_key, server_handshake_key, server_handshake_IV. The client now sends its next UDP Datagram through the VPN channel consisting of the client_initial_ack packet and the client_handshake_ack packet. This is encrypted with the client_handshake_key. The server now calculates its 1-rtt keys. This uses the shared_secret and SHA256Hash(clientHello + serverHello + serverHandshake + serverHandshake_finished) in key derivation operations to produce server_1rtt_key, client_1rtt_key. The client now calculates its 1-rtt keys. This uses the shared_secret and SHA256Hash(clientHello + serverHello + serverHandshake + serverHandshake_finished) in key derivation operations to produce server_1rtt_key, client_1rtt_key. The client now sends another UDP Datagram through the VPN channel including the client_handshake_confirmation packet, encrypted with the client_handshake_key and the client_ping packet, encrypted with the client_1rtt_key.  The server now sends another UDP Datagram through the VPN channel including the server_handshake_confirmation packet, encrypted with the server_handshake_key and the server_pong packet, encrypted with the server_1rtt_key. The client now sends the data_ack packet, encrypted with client_1rtt_key. This is migrated and sent through the regular channel. The server completes by sending the server_close_connection encrypted with the client_1rtt_key through the regular channel.
